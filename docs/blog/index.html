How to Compare LLM API Costs Across 15 Providers with One Command

Mistral Large 3 just dropped 75% in price with double the context window. The best value flagship right now is $0.50/Mtok — not from the provider you'd expect. The only way to track moves like this across all providers at once is a tool that bakes the data in.
May 9, 2026 · llm-prices · 6 min read

The problem

Every LLM provider prices their API differently, and prices change fast. Recent examples:

  • Mistral Large 3 cut its price 75% (from $2.00 to $0.50/Mtok input) and doubled context to 262k — making it the best-value "large" flagship in the market
  • Llama 4 Scout on Groq: $0.11/$0.34 per Mtok — a capable Llama 4 model at near-free prices
  • Groq, Together AI, Fireworks AI, SambaNova, and Cerebras host the same open-weight models at different prices; the gap is often 2–5×
  • Perplexity's Sonar models include a per-request search fee on top of token costs — easy to miss
  • Amazon Bedrock Nova Micro at $0.035/$0.14 per Mtok is one of the cheapest text models available
  • AI21's Jamba models offer 256k context at $0.20/$0.40 per Mtok using a hybrid SSM+Transformer architecture

There are now 15 major providers to track. Comparing them means visiting 15 different pricing pages, doing your own math for your token counts, and updating your spreadsheet every time something changes.

The solution: llm-prices

llm-prices is a zero-dependency Python CLI that bakes in pricing data for all major providers and does the math locally. No API key, no network calls at runtime, stdlib only.

# install in seconds pip install llm-prices # or: pipx install git+https://github.com/benbencodes/llm-prices

Here's a walkthrough of the six main commands.

Filter by provider or search by name:

2. Calc: what will this specific call cost?

$ llm-prices calc gpt-4o --in 10000 --out 2000 Model : gpt-4o (OpenAI) Tokens : 10,000 in / 2,000 out Rate : $2.5/Mtok in, $10.0/Mtok out Cost : $0.0250 in + $0.0200 out = $0.0450 total

Use your app's actual token counts to get a concrete cost estimate. JSON output for scripting:

$ llm-prices calc claude-sonnet-4-6 --in 5000 --out 1000 --json { "model": "claude-sonnet-4-6", "provider": "Anthropic", "input_tokens": 5000, "output_tokens": 1000, "input_cost_usd": 0.015, "output_cost_usd": 0.015, "total_cost_usd": 0.03 }

188× cheaper. Qwen3 235B-A22B via Together AI costs $0.0016 vs Claude Sonnet's $0.0300 for the same 5k/1k token call. For batch jobs or internal tooling, it's worth evaluating open-weight models before defaulting to a frontier API.

4. Budget: how many calls per dollar?

Given a budget and your typical token counts, which models can you run at scale?

Budget $1 buys you 8,000 Llama calls or 13 Claude Opus calls. That's the concrete number you need for cost planning.

5. Top: find the cheapest models for your workload

6. Providers: quick overview

$ llm-prices providers Available providers: AI21 (2 models) Anthropic (8 models) Bedrock (5 models) Cerebras (3 models) Cohere (3 models) DeepSeek (2 models) Fireworks (6 models) Google (6 models) Groq (7 models) Mistral (7 models) OpenAI (13 models) Perplexity (4 models) SambaNova (5 models) Together (7 models) xAI (2 models)

Three providers worth highlighting:

Mistral Large 3 (mistral-large-3) dropped from $2.00/$6.00 to $0.50/$1.50 per Mtok — a 75% price cut with the context window doubling to 262k. It's now priced close to GPT-4o-mini while offering flagship-class performance. Worth re-running your cost estimates if you ruled Mistral out before.

SambaNova runs on custom RDU (Reconfigurable Dataflow Unit) silicon for ultra-fast inference. They offer Llama 4 Maverick at $0.63/$1.80 per Mtok; Groq offers the same model at $0.20/$0.60 — worth comparing if latency vs. cost is your trade-off.

AI21 Labs uses a hybrid SSM+Transformer architecture (Jamba) that handles 256k context efficiently. Jamba Mini at $0.20/$0.40 with 256k context is one of the best deals for long-document workloads.

Real-world example: planning a consumer AI app

You're building an app that processes user questions. Each request is ~1,500 input tokens (system prompt + message) and ~300 output tokens.

$ llm-prices budget 100.00 --in 1500 --out 300

On $100:

  • Groq Llama 3.1 8B: ~400,000 calls
  • Gemini 2.5 Flash: ~200,000 calls
  • GPT-4o-mini: ~125,000 calls
  • GPT-4o: ~5,500 calls
  • Claude Opus 4: ~800 calls

The gap between cheapest and most expensive is 500×. For a consumer app at scale, that difference is the line between profitable and unprofitable.

Comparing the same model across inference providers

One often-overlooked optimization: the same open-weight model runs on multiple inference hosts at different prices.

$ llm-prices compare deepseek-reasoner deepseek-r1-together \ deepseek-r1-fw --in 5000 --out 2000

This compares DeepSeek R1 on DeepSeek's direct API ($0.55/$2.19 per Mtok), Together AI ($3.00/$7.00), and Fireworks ($3.00/$7.00) — pick the cheapest option or the one with better availability for your region.

Use as a Python library

from llm_prices import calculate_cost, MODELS # Calculate exact cost result = calculate_cost("gemini-2.5-flash", input_tokens=10_000, output_tokens=3_000) print(f"${result['total_cost_usd']:.4f}") # $0.0033 # Find all models under $0.50/Mtok input price cheap = {name: info for name, info in MODELS.items() if info["input_per_mtok"] < 0.50} # Get all Together AI models together = {name: info for name, info in MODELS.items() if info["provider"] == "Together"}
